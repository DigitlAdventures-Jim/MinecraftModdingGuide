Create Armor

You will want to create an armor texture and armor items using this guide. Create Armor Texture

In this guide I will be making a new dirt armor, but students should design their own ideas.

Add Armor Set Textures

  1. Navigate to MDKExample/src/main/resources/assets.dm/textures/models/armor.
  2. Drag your armor textures into the armor folder either on top of any of the files in the folder, or on the actual folder.
  3. Make sure your armor ends in _layer_1 and _layer_2 so minecraft knows which one to use as the top and bottom layer.

Add Armor Set Items

  1. Navigate to MDKExample/src/main/resources/assets.dm/textures/items.
  2. Drag your four armor item textures into the items folder either on top of any of the files in the folder, or on the actual folder.

Armor Item Models

  1. Navigate to MDKExample/src/main/resources/assets.dm/models/items.
  2. to make adding a model file more quickly select the digi_food.json file, press cmd+c and then cmd+v to copy and paste it.
  3. The digi_food.json file is the most basic model file, there are many other similar json model files that can be copied as well.
  4. Rename the file to match your new armor item, only use lowercase letters and underscores instead of spaces. This guide uses dirt_helmet
  5. double click into this new dirt_helmet.json file and change the texture location to match your new item.
    6. {
     "parent": "item/generated",
     "textures": {
         "layer0": "dm:items/dirt_helmet"
     }
  }
  6. You will do this exact process for each armor item (helmet, chestplate, leggings and boots)

Item Lang

  1. Navigate to MDKExample/src/main/resources/assets.dm/lang/en_us.lang
  2. Add in a new line of code at the bottom
  3. type "Item.dirt_helmet.name= Dirt Helmet"
  4. Everything after the = sign in this line of code will show up as the name of this item in minecraft
  5. This is only the display name, we will still use dirt_helmet as name we refer to the items resrouces.
    6.   item.dirt_helmet.name=Dirt Helmet
  item.dirt_chestplate.name=Dirt Chestplate
  item.dirt_leggings.name=Dirt Leggings
  item.dirt_boots.name=Dirt Boots

Initialize

Armor Material

Armor materials are objects we attach to armor items so that they connect to our texture and to different stats we want to give to the armor.

  1. Navigate to /MDKExample/src/main/java/com/digitaladventures/digimod/init/DigiItems.java
  2. Scroll to the top where it says "//Materials"
  3. Copy an ArmorMaterial that already exists, this will make coding much quicker.
  4. Paste it just below but make some changes to make it match your new armor.
  5. 	public static final ArmorMaterial ARMOR_MATERIAL_DIRT = EnumHelper.addArmorMaterial("armor_material_dirt", Reference.MOD_ID + ":dirt", 14,
			new int[] {2, 5, 7, 3}, 10, SoundEvents.ITEM_ARMOR_EQUIP_DIAMOND, 0.0F);
  6. The parameters for addArmorMaterial are listed as the following
    1. String: name of the resource in minecraft commands
    2. String: resource location for the armor textures
    3. Integer: durability
    4. Array of Intergers: damage reduction for helmet, chestplate, leggings, boots.
    5. Integer: enchantability
    6. SoundEvent: Sound played when put onto player
    7. Float: toughness multiplier

Armor Items

Armor items are initialized much like any other item except that they have three extra parameters, armor material, layer and armor slot.

  1. Copy and paste the 4 Armor items already being initialized to speed up the process.
  2. You will need to make a few changes for these items to work with your new dirt armor.
  3. 	  public static final Item DIGI_HELMET = new ArmorBase("digi_helmet", ARMOR_MATERIAL_DIRT, 1, EntityEquipmentSlot.HEAD);
	 public static final Item DIRT_CHESTPLATE = new ArmorBase("dirt_chestplate", ARMOR_MATERIAL_DIRT, 1, EntityEquipmentSlot.CHEST);
	 public static final Item DIRT_LEGGINGS = new ArmorBase("dirt_leggings", ARMOR_MATERIAL_DIRT, 2, EntityEquipmentSlot.LEGS);
	 public static final Item DIRT_BOOTS = new ArmorBase("dirt_boots", ARMOR_MATERIAL_DIRT, 1, EntityEquipmentSlot.FEET);
  4. The first parameter of each item should just be the resource/texture name.
  5. The material parameter will be the one we just created.
  6. The layer of 1 means the armor will stick out of your player like a helmet or shoulderpads, layer of 2 means it will be flush to the player like a t-shirt or pants.
  7. The final new parameter is the slot the armor belongs to.
