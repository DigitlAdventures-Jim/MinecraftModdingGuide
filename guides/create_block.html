Create Block

Block Textures

  1. Navigate to MDKExample/src/main/resources/assets.dm/textures/blocks.
  2. Drag your block texture into the blocks folder either on top of any of the files in the folder, or on the actual folder.

Block Item Model

The item texture for a block is what the block looks like while being held.

  1. Navigate to MDKExample/src/main/resources/assets.dm/models/item.
  2. to make adding a model file more quickly select the digi_block.json file, press cmd+c and then cmd+v to copy and paste it.
  3. The digi_block.json file is the most basic item model file for blocks.
  4. Rename the file to match your new block, only use lowercase letters and underscores instead of spaces. This guide uses my_block but you should use your own unique item name
  5. double click into this new my_item.json file and change the texture location to match your new block. This code simply points to the block model as the one it should use for when the block is being held like an item.
    6. {
    {
       "parent": "dm:block/my_block"
    }

Block Block Model

  1. Navigate to MDKExample/src/main/resources/assets.dm/models/block.
  2. To make adding a model file faster select the digi_block.json file, press cmd+c and then cmd+v to copy and paste it.
  3. The digi_block.json file is the most basic block model file for blocks.
  4. Rename the file to match your new block, only use lowercase letters and underscores instead of spaces.
  5. double click into this new my_block.json file and change the texture location to match your new block.
  6. block/cube_all is the key to this script as it makes the texture for our block the same on all sides.
    7. {
    {
       "parent": "block/cube_all",
       "textures": {
           "all": "dm:blocks/digi_block"
       }
    }

Add a BlockState file

Block states are ways a block can look differently in different situations in minecraft. Doors open or closed, levers up or down, fences facing north, south, east or west.

  1. Navigate to MDKExample/src/main/resources/assets.dm/lang/en_us.lang
  2. Copy and past the digi_block.json file, or create a new .json file with the code below.
  3. this json markup code is to just display the block without any varients.
    4.  {
    "variants": {
        "normal": { "model": "dm:my_block" }
    }
  }

Block Lang

  1. Navigate to MDKExample/src/main/resources/assets.dm/lang/en_us.lang
  2. Add in a new line of code at the bottom
  3. type "Item.my_item.name= My New Item"
  4. Everything after the = sign in this line of code will show up as the name of this item in minecraft
  5. This is only the display name, we will still use my_item as name we refer to the item as in our code
    6.   tile.my_block.name=My New Block

Initialize

  1. Navigate to /MDKExample/src/main/java/com/digitaladventures/digimod/init/DigiBlocks.java
  2. Scroll to where it says "//New Blocks"
  3. Add this new line of Java Code "public static final BlockBase MY_BLOCK = new BlockBase("my_block");"
  4. In this code we are creating a new BlockBase called MY_BLOCK which is an extension of the standard minecraft Block
  5. BlockBase holds all of the information for adding this block into minecraft and connecting the texture and model files to this item.
  6. Public static final are keywords which describe how this new instance or clone of the BaseBlock can be interacted with.
