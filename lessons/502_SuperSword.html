  • Skill Level Basic

Introduction

In this project we will be creating a new sword that will have special abilities. We will learn how to add a new texture into our game as well as learn a few functions and how to comment and uncomment code.

Activity

In this activity student should learn how parameters work in functions

  1. Functions or Methods are ways that we can tell objects or things to do a task or instruction in Java.
  2. Have each student create a single word function on the board like "Jump()", "Spin()" or "Draw()".
  3. Next students should do each of the functions.
  4. Then inside of the "()" write down a parameter for each function. For example. Jump(height), Spin(rotations), Draw(shape)
  5. Make each student then fill in the blanks and choose a number/word for each function like Jump(10 feet), Spin(2 times) or Draw(circle)
  6. Have students do each of the functions again.

Customize Sword Abilities

Navigate to and open src/main/java/com/digitaladventures/digimod/items/DigiSuperSword.java

There should be a method hitEntity which is activated when an entity is hit with this sword.

We will be adjusting the code within this method, you'll notice each of the functions is green. This code is commented, meaning it's disabled and is for the computer to ignore. '//' can comment out 1 line, if you want to comment out more lines you can use '/*' and at the end of the comment '*/'

  • target.setFire() requires a number for how many seconds to set the enemy/target on fire.
  • attacker.playSound() requires a sound event which can be accessed by typing 'SoundEvents.'
  • target.setVelocity() has 3 parameters, the x,y and z speed of the enemy. 0,5,0 will launch them straight upwards.
  • target.addPotionEffect() is a bit more complex as you'll need to create a new potion each time this occurs. You can customize the MobEffects. portion of this code.
  • the last chunk of code creates an explosion, which you can enable by removing the /* and */

Advanced

Add your own functions by typing in 'target.' or 'attacker.' and search through the list that pops up and try to write in the correct parameters.
